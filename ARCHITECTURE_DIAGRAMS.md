## ğŸ“ Key Concepts Explained

### 1. **Flow vs StateFlow**
```
Flow<T>
- Cold stream (starts on collection)
- Multiple collectors get separate streams
- Good for one-time operations

StateFlow<T>
- Hot stream (always active)
- All collectors share same stream
- Caches latest value
- Perfect for UI state
```

### 2. **callbackFlow**
```
callbackFlow {
  // Convert callback-based API to Flow
  listener = addListener { data ->
    trySend(data)  // Emit to Flow
  }
  
  awaitClose {
    removeListener(listener)  // Cleanup
  }
}
```

### 3. **collectAsState()**
```
// Converts StateFlow to Compose State
val items by viewModel.items.collectAsState()

// Whenever StateFlow emits new value:
// 1. Compose State updates
// 2. Composable function reruns
// 3. UI displays new data
```
